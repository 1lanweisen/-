一、检索计划与数据可行性核查结果
1) 检索计划
本次核查分四层推进：
第一层，核验你草案所依赖的核心公开证据链是否真实存在：Shintani 2024 正文、其 figshare 补充数据、GEO/SRA 页面是否仍公开可访问。Shintani 2024 的确公开发表，声明纳入了 5 类 ABA 相关胁迫、216 个 treated-control paired datasets，并把 RNA-seq 元数据与若干补充表放在 figshare v6；figshare 页面也明确列出了补充表 1–15 的内容。(Frontiers)
第二层，做你要求的数据可行性核查：逐项确认公开数据的文件构成、字段层级、样本映射、格式、以及是否足以支撑“contrast-level 不确定性估计”。这里重点不是“能不能下载”，而是“能不能在你的主方案里合法入链”。NCBI 官方说明表明，SRA 的公共对象层级是 STUDY/SAMPLE/EXPERIMENT/RUN；RUN 对应实际测序数据文件，GEO Sample 页面又能回链到 Series、BioSample 和 SRA，因此“从公开页面重建下载与映射链”在结构上是可行的。(NCBI)
第三层，查找真正的断点。断点已经出现两个：其一，Shintani 的 Supplementary Table 1 只明确写了 SRA study ID、run ID、tissue、treatment type、treatment time、library type，但没有直接声明 treated-control 的 pair_id，也没有直接给 replicate grouping；其二，至少有部分 GEO 样本是“先把 3 个生物学重复 pooled，再测序”，这类样本无法支撑你要求的 contrast-level biological SE。也就是说，Shintani 论文中的“216 paired datasets”不能直接等价成你方案里的“216 个可估计 SE 的 eligible contrasts”。(Figshare)
第四层，核查解释层资源是否真实公开并足够成熟。Arabidopsis DAP-seq 529 TF cistrome、ChIP-Hub 植物 regulome 平台、2023–2025 的 Arabidopsis 单细胞/单核与空间图谱都公开可用，因此“上游 TF/cistrome + 组织/细胞语境解释”这一层是可行的，但它只能作为解释层，不能反向改写主结果标签。(PubMed)
2) 数据可行性核查结论
总结判断
结论：条件可行，但不是整包 216 对都可直接入链。
能成立的前提是：先做一次严格的 eligible contrast freeze，只保留同时满足以下条件的 contrasts：公开 raw FASTQ 可下载；treated 与 control 的配对关系可由 GEO/SRA/原文回溯；生物学重复不是 pooled 后单次测序；组学对象是 Arabidopsis thaliana 野生型；组织与时间信息足以与对照匹配。只要其中任一条件不成立，该 contrast 就必须剔除。这个结论与近年的 RNA-seq 元分析综述、公共元数据完整性讨论是一致的：公共 RNA-seq 再利用的核心瓶颈不只是批次差异，而是样本元数据、实验设计和依赖结构能否被可靠重建。(ScienceDirect)
已确认可行的部分
Shintani 2024 正文与 figshare 补充数据真实存在，且 figshare v6 明确可下载；Supplementary Table 1 提供了足以追到 SRA/GEO 的最小索引字段，Supplementary Table 2–8 提供了作者处理后的 TPM、TN-ratio、TN-score、ABA count 和 DESeq2 结果。公开页还显示整个 figshare 数据集大小约 197.51 MB。(Frontiers)
GEO Sample 页面公开暴露了许多你需要的关键字段，例如 sample title、organism、characteristics、tissue、treatment 描述、Series accession、BioSample、SRA accession、以及“Raw data are available in SRA”。SRA 官方又说明 RUN 是数据文件清单，paired-end 的 forward/reverse 必须在同一 RUN 中登记。因此，从 GEO/GSM → SRA/SRX/SRR → FASTQ 的下载链是成立的。(NCBI)
统一重建 gene-level 输入矩阵也是成立的。Salmon、tximport、DESeq2 仍是主流且相互兼容的路径；Arabidopsis 当前仍可获得 TAIR10/Araport11 注释，Araport11 是 TAIR10 参考基因组上的完整重注释。(Nature)
已确认的关键断点
断点 1：pair_id 不是直接公开字段。
Supplementary Table 1 的说明只写到 study ID、run ID、tissue、treatment type、treatment time、library type，没有把 treated-control 显式编码成 pair_id。你的主方案若要求“treated-control 可配对且能估计 contrast-level uncertainty”，就必须再到 GEO Sample/Series 页面与原始论文逐项补 pair 关系，不能把 Shintani 的 216 pairs 直接照搬。(Figshare)
断点 2：部分样本是 pooled biological replicates。
例如 GEO 样本 GSM1936762 直接写明 “the three biological replicates ... were pooled into one well”。这类样本最多能参与作者式的 TN-ratio/TN-score 统计，却不能进入你要求的“contrast-level biological SE”主线。(NCBI)
断点 3：GEO/BioSample 属性是半结构化甚至自由文本。
NCBI 说明 BioSample 既有 harmonized attributes，也允许 submitter 自定义字段；这意味着 tissue、time、treatment、replicate 等经常不是统一枚举值，必须手工标准化。(NCBI)
对你当前研究目标的直接影响
因此，你想做的不是“直接升级 Shintani 的再分析”，而是要把研究对象从“216 个作者定义的 pairs”收紧成“能支撑 effect size + uncertainty + dependence 的 eligible contrasts”。这一步是必要的，否则后面的 SE、依赖结构和类内元分析都会失真。近年的植物 RNA-seq 元分析综述同样强调，效应量与研究间异质性是决定结果可重复性的关键，不应只做阈值式 DEG 叠加。(ScienceDirect)

二、完整实验方案（中文）
简版一页总览
0) 总体大纲（只允许一条主线）
唯一科学问题： 在 Arabidopsis thaliana 野生型的五类 ABA 相关胁迫公开 RNA-seq 中，若只保留“原始 FASTQ 可回溯、treated-control 可严格配对、且能估计 contrast-level 不确定性”的 eligible contrasts，是否存在一个在五类处理间都稳定成立的 shared core stress-response genes 集合，以及一个相对其余四类处理表现出稳定偏置的 dehydration-biased genes 集合；若存在，这两类基因分别落在哪些 Arabidopsis 上游 TF/cistrome 与组织/细胞语境中。这个方案的主叙事是回答“共同响应与脱水偏置响应是否真实存在、由谁调控、处于什么细胞语境”，不是方法学研究，也不以方法优化为主叙事。这一问题边界与既有“common vs unique stress response”研究相接，但不等于复做原文，因为你的入链标准改成了“effect size + uncertainty + dependence”。(PLOS)
一页式主线
先从公开证据链中冻结 eligible contrasts，再从原始 FASTQ 统一重建 gene-level count 矩阵；随后在 study 内提取每个 contrast 的基因效应量与 SE，并显式编码 shared control / same study / nearby time 等依赖结构；接着在五个 treatment class 内分别做依赖感知的随机效应元分析，得到 gene × treatment 的 pooled effect 矩阵；再仅判定两个主终点：shared core 与 dehydration-biased；最后用 Arabidopsis DAP-seq、ChIP-Hub 和单细胞/空间图谱解释这两类基因的上游调控与组织/细胞定位，但解释层不回写主结果标签。(PubMed)

1.2 各数据源的文件构成、结构、风险与可运行性
数据源 A：Shintani 2024 + figshare v6
•	文件构成：正文 + figshare Supplementary Table 1–15，另有补充图。Supplementary Table 1 是 curated datasets metadata；Table 2–4 是 TPM、TN-ratio、TN-score；Table 5–6 是 ABA 的 read count 与 DESeq2 结果；Table 7–15 是各处理或跨处理的 DEG/富集结果。(Frontiers)

•	数据类型与格式：figshare 页面未逐一展开文件扩展名，但描述显示包含表格型补充文件；可视为元数据/表达矩阵/结果表。(Figshare)
•	缺失风险：不能仅靠该表完成 pair 重建与 replicate 合法性判定。
•	可运行性结论：可作为 cohort 索引与核查辅助，不能直接作为主分析输入。(Figshare)
数据源 B：GEO Sample/Series 页面
•	文件构成：Sample 页面、Series 页面、补充文件入口、SRA Run Selector 入口。
•	数据结构：常见字段含 sample title、source name、organism、characteristics、treatment protocol、growth protocol、library strategy/source/selection、instrument、series accession、BioSample、SRA accession。部分样本明示 replicate 信息，部分样本明示“mock control”。(NCBI)
•	数据类型与格式：HTML 页面 + SOFT/MINiML；可回链到 SRA raw data。(NCBI)
•	缺失风险：属性是半结构化文本，不同 study 的字段名和写法不统一；有的样本处理说明只在正文里完整。
•	可运行性结论：可支撑 pair 重建与 replicate 核查，但需要人工标准化。(NCBI)
数据源 C：SRA/BioSample
•	文件构成：STUDY（SRP）、SAMPLE（SRS/BioSample）、EXPERIMENT（SRX）、RUN（SRR）。
•	数据结构：NCBI 说明 RUN 是测序数据文件的 manifest，EXPERIMENT 记录 library / layout / instrument 等技术属性；paired-end 文件必须成对登记在同一 RUN。(NCBI)
•	数据类型与格式：原始序列可导出 FASTQ；元数据来自 SRA/BioSample 记录。(NCBI)
•	缺失风险：SRA 保证技术层级，不保证生物学 pairing 语义完整。
•	可运行性结论：足以支持 raw FASTQ 下载与技术元数据核查，但不能单独解决 treated-control 语义配对。(NCBI)
数据源 D：Arabidopsis 参考注释
•	文件构成：TAIR10 assembly、Araport11 annotation、cDNA/GFF3。
•	数据结构：TAIR10 是当前参考组装；Araport11 是基于 TAIR10 的完整重注释。(PubMed)
•	数据类型与格式：FASTA / GFF3 / cDNA。
•	缺失风险：若沿用作者旧版 Ensembl Plants release 53，再与新 atlas/cistrome 对接时，可能产生 ID 映射损失。
•	可运行性结论：可运行，但必须预先固定 annotation 版本。(PubMed)
数据源 E：解释层资源（DAP-seq / ChIP-Hub / atlas）
•	文件构成：DAP-seq Arabidopsis cistrome；ChIP-Hub 的 reanalyzed regulome/epigenome；Arabidopsis 单核/单细胞/空间 atlas。
•	数据结构：DAP-seq 给出 TF binding peaks/motifs；ChIP-Hub 汇总 ChIP-seq/DAP-seq/DNase/ATAC 等植物 regulome 数据；2025 life-cycle atlas 跨 10 个发育阶段、>400k nuclei，并提供 browser/CELLxGENE 访问；2023 seedling atlas提供细胞型分辨率的 Arabidopsis seedling 参考。(PubMed)
•	数据类型与格式：平台化浏览数据、峰集、单细胞/空间表达矩阵。
•	缺失风险：这些资源主要用于“解释层”，不是同一胁迫条件下的 bulk RNA-seq 直接验证。

2) 实验主线步骤（连续、紧凑、必要）
Step 1：公开 cohort 复核与 eligible contrast 冻结
步骤目标（科学问题导向）
把“Shintani 定义的 216 paired datasets”收紧为“可支撑 effect size + uncertainty + dependence 的 eligible contrasts”，从源头保证后面回答的是真实稳定的共同响应与脱水偏置响应，而不是元数据错误叠加的假阳性。(Frontiers)
输入数据来源
来自原始数据：Shintani 2024 正文、figshare Supplementary Table 1、GEO Sample/Series 页面、原始论文方法部分。(Frontiers)
核心方法
逐条重建 study_id–run_id–treated/control–replicate–tissue–time 的关系；显式创建 pair_id；剔除 pooled biological replicates、无法确认野生型、无法确认对照关系、组织或时间不匹配、或无 raw FASTQ 的条目；按五类处理分别冻结 eligible contrast 清单。
输出产物
cohort_freeze.tsv、eligible_contrast_manifest.tsv、exclusion_log.tsv。

Step 3：study 内部提取 gene-level contrast effect size 与 uncertainty，并编码依赖结构
步骤目标（科学问题导向）
把每个 eligible contrast 转成可进入元分析的最小证据单元：每个 gene 的 log2FC 与 SE，并同时记录它和其他 contrasts 的非独立性。
输入数据来源
来自前一步结果：gene_count_matrix.tsv、design_matrix.tsv、eligible_contrast_manifest.tsv。
核心方法
在每个 study 内按 treated-control 配对做差异表达分析；输出 gene-level log2FC 与标准误；同时为 shared control、same study、same protocol、nearby time、same tissue 等关系建立 dependency coding；凡不能稳定估计 biological SE 的 contrast 直接标记 ineligible。

Step 4：在五个 treatment class 内做依赖感知的随机效应元分析
步骤目标（科学问题导向）
不是问“某基因在某个单 study 里变不变”，而是问“在 ABA、Salt、Dehydration、Mannitol、Cold 各自这一类胁迫下，它的总体方向和不确定性是什么”。
输入数据来源
来自前一步结果：gene_effects.tsv、dependency_map.tsv、final_eligible_contrasts.tsv。
核心方法
按 treatment class 分层，对每个 gene 聚合多个 contrast 的 effect size；模型中显式考虑研究间异质性与 effect-size dependence；对 shared control 或同 study 多 contrast 的相关性使用多层模型或稳健方差估计，不假定完全独立。

Step 5：构建 gene × 5-treatment 证据矩阵，只判定两个主终点
步骤目标（科学问题导向）
将所有信息压缩成一个唯一终点判定层：哪些基因是五类处理中都稳定成立的 shared core，哪些基因表现为 dehydration-biased。
输入数据来源
来自前一步结果：meta_effects_by_treatment.tsv、robustness_screen.tsv。
核心方法
构建 gene × five-treatment pooled effect matrix；先判定“跨五类方向稳定且证据一致”的 shared core；再在剩余基因中判定“对 dehydration 的 pooled effect 相对其余四类持续偏置”的 dehydration-biased；其余全部留在 background，不扩展第三主终点。

Step 6：对两类固定结果做 TF/cistrome 与组织/细胞语境解释
步骤目标（科学问题导向）
回答最后一个生物学问题：这些 shared core 与 dehydration-biased 基因更可能由哪些上游调控程序驱动，又更像落在什么组织/细胞语境中。
输入数据来源

核心方法

3) 风险与审稿人攻击点
风险 1：treated-control 配对关系被误判
这会直接让 effect size 失效，因为错误对照会把“处理效应”与时间/组织/培养差异混在一起。规避方向是：pair_id 必须由 figshare 索引、GEO Sample/Series、原始论文三方交叉确认；任何一条证据链断裂的 contrast 直接排除。公共转录组元数据复用最常见的问题就是样本元数据不充分或不一致。(Figshare)
风险 2：把 pooled biological replicates 当作可估计 SE 的独立重复
这会让标准误被系统性低估，从而把不确定结果伪装成稳定信号。规避方向是：凡 GEO 或原文写明 pooled biological replicates 的样本，一律不进入 Step 3 主线。GSM1936762 已经是明确实例。(NCBI)
风险 3：shared control / 同一 study 多 contrasts 的依赖性被当成独立
这会让 treatment-class 元分析的置信区间过窄，shared core 被夸大。规避方向是：必须显式编码 dependency_group，并使用多层随机效应或稳健方差方法。(PubMed)
风险 4：不同组织与时间点的异质性压过处理主效应
这会使所谓“shared core”其实只是某些高频组织或时间窗口的共性，而不是处理共性。规避方向是：冻结 tissue/time 字段，要求 treated-control 在组织和时间上可匹配；同时在类内元分析后保留异质性摘要，异质性过高者不进入 headline results。(Frontiers)
风险 5：注释版本不一致导致 gene ID 映射丢失
这会让 bulk RNA-seq 主结果无法稳定映射到 DAP-seq、ChIP-Hub 和 atlas，解释层因此漂移。规避方向是：一开始就锁定 TAIR10/Araport11 映射规则，所有下游资源统一用同一 gene ID 空间。(PubMed)
