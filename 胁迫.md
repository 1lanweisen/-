## 一页式总览

**主线问题**
在 Arabidopsis thaliana 野生型的五类 ABA 相关胁迫公开 RNA-seq 中，只保留“原始 FASTQ 可回溯、treated-control 可严格配对、且能估计 contrast-level 不确定性”的 eligible contrasts 后，是否仍存在一组**跨五类处理稳定复现的 shared core genes**，以及一组**相对其余四类处理表现出稳定脱水偏置的 dehydration-biased genes**；若存在，这两类基因更可能对应哪些上游 TF/cistrome 与哪些组织/细胞语境。Shintani 2024 的原始公开证据链确实覆盖五类处理和 216 个 treated-control pairs，但正文同时表明公开索引只直接给到 study/run/tissue/treatment/time/library 等字段，因此主问题必须建立在“先冻结真正可运行 contrasts”之上，而不能直接把作者级 pairs 原封不动搬进主链。([Frontiers][2])

**核心假设**
如果一个响应只在元数据模糊、共享对照未处理、或 pooled biological replicates 混入时才显著，那么它不应被视为稳健科学结论；相反，在统一注释、统一定量、显式处理 uncertainty 与 dependence 后仍能保留下来的信号，才更接近真正的共同胁迫响应与脱水偏置响应。最新植物 RNA-seq 元分析综述也强调，可靠的跨研究整合依赖标准化预处理、统一注释与异质性处理，而不是简单叠加 DEG 名单。([ScienceDirect][1])

**结论路径**
前置准备只做合法入链判定，不产出主结果；正式主链从统一重建 gene-level 输入开始，经 study 内 effect size 与 SE 提取、类内依赖感知元分析、跨五类主终点判定，最后再把固定结果投影到 Arabidopsis DAP-seq、ChIP-Hub 和公开单细胞/单核/空间 atlas 中解释其 TF 与细胞语境。这样，主结论只由 Step3 及之后产生，而且始终围绕同一个科学问题。([Rutgers University][3])

## 前置准备（不计入主实验链）

### Step1 数据可得性核查

**数据集文件构成**
可直接获得的公开对象包括：Shintani 2024 正文、figshare 补充表、GEO Sample/Series 页面、SRA/BioSample 元数据、SRA 原始测序文件、Arabidopsis 参考注释，以及解释层所需的 DAP-seq、ChIP-Hub 与公开 atlas。([Frontiers][2])

**数据结构**
源 cohort 的最小可追踪索引是 study_id、run_id、tissue、treatment_type、treatment_time、library_type；GEO/SRA 进一步提供 series、biosample、experiment 与 run 的映射；BioSample 同时允许 recognized attributes 与 submitter 自定义属性，因此 tissue、time、treatment、replicate 常不是统一枚举值。([Frontiers][2])

**数据组成**
公开来源显示，原始作者纳入的是 Arabidopsis 五类 ABA 相关处理的 treated-control paired datasets；处理类别对应 ABA、盐、脱水、甘露醇渗透和低温，组织覆盖 seedlings、leaves、roots、shoots、bud 等。这个作者级 cohort 真实存在，但还不是你主实验链中的合法样本集合。([Frontiers][2])

**呈现形式**
原始测序层为 SRA/FASTQ；样本与系列层为 GEO 页面记录；补充结果层为 figshare 表格；注释层为 Arabidopsis 参考序列与注释；解释层为 regulome/atlas 资源。([Frontiers][2])

**对外来研究者的可得性判断**
对外部研究者**可直接获得**的是 paper、figshare 补充表、GEO/SRA 页面、原始 FASTQ、参考注释、DAP-seq、ChIP-Hub 与 atlas；**不能直接获得但可重建**的是 pair_id、replicate_group、dependency_group 和统一 design matrix；**若 treated-control 关系或重复结构在公共入口中不可验证，则该 contrast 对外部研究者不可复现，也不得入主链**。([NCBI Insights][4])

### Step2 元数据完整性核查与可运行性门槛判定

**最小必需字段**
主链最低要求字段为：study_id、series_id、biosample_id、experiment_id、run_id、genotype、treatment_class、control_flag、pair_id、replicate_id、pooled_replicate_flag、tissue、time_point、library_layout、fastq_path；以及后续重建出的 gene_id、count、log2FC、SE、dependency_group。缺失的不是一般注释，而是会直接导致主链断裂的结构性字段。 ([NCBI][5])

**可运行性门槛**
只有同时满足以下条件的 contrast 才允许进入 Step3：原始 FASTQ 可获得；treated-control 关系可回溯；genotype 可确认是野生型；组织与时间对照可匹配；重复结构可识别且不是 pooled biological replicates；后续能映射到统一 Arabidopsis gene ID 空间。任一条件不满足，直接停在前置准备。([Frontiers][2])

**关键断点**
缺 `run_id/fastq_path`，Step3 无法统一重建输入矩阵；缺 `pair_id/control_flag`，Step4 无法计算 contrast effect；缺 `replicate_id` 或 `pooled_replicate_flag` 不清，Step4 无法得到可信 biological SE；缺 `tissue/time/genotype`，Step5 无法把 pooled signal 解释为处理类效应；缺统一 `gene_id` 映射，Step7 无法合法投影到 DAP-seq、ChIP-Hub 与 atlas。GEO 样本 GSM1936762 已公开写明其生物学重复在测序前被 pooled，这类样本不能进入需要 biological SE 的主链。([NCBI][6])

## 正式主实验链（Step3起，逐步展开）

### Step3 统一重建 eligible contrasts 的 gene-level 输入矩阵

**步骤目标（科学问题导向）**
把所有合格 contrasts 放到同一注释与同一表达量坐标系中，为后续比较“共同响应”和“脱水偏置响应”提供可比输入。

**输入数据来源（明确来自原始数据或前一步哪个文件）**
来自 Step2 冻结后的 `eligible_contrast_manifest.tsv`、对应 `run_id` 的原始 FASTQ，以及 Arabidopsis 统一参考注释与转录本序列。

**方法（只写方法，不写参数）**
对原始 FASTQ 做统一质控与清洗；采用同一转录本定量流程重建 transcript abundance；再汇总到 gene-level counts；固定 TAIR10/Araport11 映射规则；生成统一的 sample design 表。

**输出产物（文件名或结果对象）**
`sample_manifest.tsv`、`qc_summary.tsv`、`gene_count_matrix.tsv`、`design_matrix.tsv`、`annotation_lock.tsv`

**本步骤依据论文 DOI（该步骤实际采用的方法来自哪篇论文）**
10.1093/bioinformatics/bty560；10.1038/nmeth.4197；10.12688/f1000research.7563.2。([OUP Academic][7])

**与前一步衔接依据 DOI（说明这两步为何这样连接）**
10.1016/j.cpb.2025.100523；10.1111/tpj.13415。前者说明跨研究 RNA-seq 元分析需要标准化预处理与统一注释，后者提供 Arabidopsis 统一注释基线。([ScienceDirect][1])

### Step4 在 study 内提取 gene-level effect size 与 uncertainty，并显式编码 dependence

**步骤目标（科学问题导向）**
把每个 eligible contrast 转成真正可进入元分析的最小证据单元：每个 gene 的效应方向、效应大小与不确定性，同时明确它与其他 contrasts 的非独立关系。

**输入数据来源（明确来自原始数据或前一步哪个文件）**
来自 Step3 的 `gene_count_matrix.tsv`、`design_matrix.tsv` 和 Step2 的 `eligible_contrast_manifest.tsv`。

**方法（只写方法，不写参数）**
在 study 内按 treated-control 设计进行差异表达分析，提取 gene-level log2FC 与标准误；同时为 shared control、same study、same tissue、nearby time 等关系建立 dependency coding；凡无法稳定估计 biological SE 的 contrast 立即剔除。

**输出产物（文件名或结果对象）**
`gene_effects.tsv`、`contrast_qc.tsv`、`dependency_map.tsv`、`final_eligible_contrasts.tsv`

**本步骤依据论文 DOI（该步骤实际采用的方法来自哪篇论文）**
10.1186/s13059-014-0550-8。([Springer][8])

**与前一步衔接依据 DOI（说明这两步为何这样连接）**
10.12688/f1000research.7563.2。因为 Step3 的 transcript-to-gene 汇总正是为后续 count-based gene-level inference 服务。([F1000Research][9])

### Step5 在五个 treatment class 内做依赖感知元分析，生成 gene × treatment 证据矩阵

**步骤目标（科学问题导向）**
回答“某基因在某一类胁迫下总体上是否稳定改变”，而不是停留在某一个 study 的差异表达。

**输入数据来源（明确来自原始数据或前一步哪个文件）**
来自 Step4 的 `gene_effects.tsv`、`dependency_map.tsv`、`final_eligible_contrasts.tsv`

**方法（只写方法，不写参数）**
按 treatment class 分层，对每个 gene 聚合多个 contrasts 的 effect size；使用随机/多层元分析框架处理研究间异质性；对 shared control 与同 study 多 contrasts 的相关性采用稳健方差估计或等价的依赖感知方案，不把它们当成完全独立证据。

**输出产物（文件名或结果对象）**
`meta_effects_by_treatment.tsv`、`heterogeneity_summary.tsv`、`gene_by_treatment_matrix.tsv`

**本步骤依据论文 DOI（该步骤实际采用的方法来自哪篇论文）**
10.1186/1471-2105-15-91；10.1002/jrsm.5；10.18637/jss.v036.i03。([Springer][10])

**与前一步衔接依据 DOI（说明这两步为何这样连接）**
10.1002/jrsm.5；10.1186/1471-2105-15-91。因为 Step4 已经给出 gene-level effect、SE 与 dependence，Step5 才能合法聚合。([PubMed][11])

### Step6 主终点判定：只判定 shared core 与 dehydration-biased 两类结果

**步骤目标（科学问题导向）**
把跨五类处理的整合证据压缩为两个唯一主终点：真正共享的 core 响应，以及真正偏向脱水的响应。

**输入数据来源（明确来自原始数据或前一步哪个文件）**
来自 Step5 的 `gene_by_treatment_matrix.tsv`、`meta_effects_by_treatment.tsv`、`heterogeneity_summary.tsv`

**方法（只写方法，不写参数）**
对五类处理的 class-level 证据实施部分合取检验，用于判定 shared core；再以 treatment class 为 moderator 进行类间比较，识别在 dehydration 中相对其余四类处理持续偏置的基因；其余基因全部保留为 background，不扩展第三主终点。

**输出产物（文件名或结果对象）**
`shared_core_genes.tsv`、`dehydration_biased_genes.tsv`、`background_genes.tsv`、`primary_conclusion_table.tsv`

**本步骤依据论文 DOI（该步骤实际采用的方法来自哪篇论文）**
10.1111/j.1541-0420.2007.00984.x；10.18637/jss.v036.i03。([ResearchGate][12])

**与前一步衔接依据 DOI（说明这两步为何这样连接）**
10.1111/j.1541-0420.2007.00984.x。因为 Step5 产出的正是五类处理层面的可组合证据，适合进入 replicability/partial-conjunction 判定。([OUP Academic][13])

### Step7 固定主终点后的解释层：TF/cistrome 与组织/细胞语境

**步骤目标（科学问题导向）**
解释 shared core 与 dehydration-biased 两类基因更可能由谁调控、出现在什么组织/细胞语境中，但不再改写主标签。

**输入数据来源（明确来自原始数据或前一步哪个文件）**
来自 Step6 的 `shared_core_genes.tsv`、`dehydration_biased_genes.tsv`；外部资源来自 Arabidopsis DAP-seq、ChIP-Hub、Arabidopsis seedlings 单细胞图谱、Arabidopsis life-cycle 单核/空间图谱及 leaf senescence atlas。([Rutgers University][3])

**方法（只写方法，不写参数）**
将 Step6 固定 gene sets 分别与 DAP-seq TF target sets、ChIP-Hub regulome targets 和 atlas-derived cell-type marker sets 做富集分析；必要时对全基因组排序证据做 enrichment 复核；最终只输出支持层级与解释优先级，不回写主终点标签。

**输出产物（文件名或结果对象）**
`tf_prioritization.tsv`、`cistrome_support.tsv`、`cell_context_support.tsv`、`integrated_interpretation_table.tsv`

**本步骤依据论文 DOI（该步骤实际采用的方法来自哪篇论文）**
10.1089/omi.2011.0118；10.1073/pnas.0506580102。([PubMed][14])

**与前一步衔接依据 DOI（说明这两步为何这样连接）**
10.1089/omi.2011.0118。因为 Step6 先固定 gene clusters，Step7 才能对这些 gene clusters 做合法解释。([PubMed][14])

## 风险与审稿人高频质疑点（仅列关键项）

1. **treated-control 是否错配**
   pair_id 不是公共现成字段，必须由 figshare 索引、GEO/SRA 页面和原文三方交叉重建；任一链断裂，该 contrast 排除。([Frontiers][2])

2. **pooled biological replicates 是否混入主链**
   凡原始页面或原文注明 pooling，就不能进入需要 biological SE 的 Step4。([NCBI][6])

3. **共享对照与同 study 多 contrasts 的依赖性是否被误当独立**
   必须在 Step4 先编码 dependence，再在 Step5 用依赖感知元分析处理。([PubMed][11])

4. **组织与时间异质性是否压过处理主效应**
   这不是后验修补，而是前置门槛；组织与时间不匹配的 contrast 在 Step2 就不能入链。([Frontiers][2])

5. **解释层是否反向定义主结论**
   不能。主终点只能由 Step6 产生；Step7 只做机制与细胞语境解释。([Rutgers University][3])

## 步骤必要性确认（逐步说明“不可缺少，也不多余”）

**Step1 不可缺少**
它回答的是“公开数据到底拿不拿得到”。没有这一步，整项研究可能从一开始就建在不存在或不可复算的数据链上。它也不多余，因为它只做可得性核查，不碰主结果。([Frontiers][2])

**Step2 不可缺少**
它回答的是“这些公开数据能不能合法进入 effect size + uncertainty 主链”。没有这一步，Step4 的 biological SE 与 Step5 的元分析会失真。它也不多余，因为它只做门槛判定，不产出科学结论。([NCBI][6])

**Step3 不可缺少**
它解决“不同 study 是否处在同一表达量坐标系”这个根问题。没有统一重建，后续 pooled signal 不可比。它也不多余，因为它承担的唯一任务就是把可运行 contrasts 放进同一 gene-level 空间。([ScienceDirect][1])

**Step4 不可缺少**
它是从“表达矩阵”走向“可元分析证据单元”的唯一桥梁。没有 gene-level effect、SE 和 dependence，Step5 无法合法聚合。它也不多余，因为它不做跨 study 结论，只做证据提取。([Springer][8])

**Step5 不可缺少**
它把单 study 证据变成 treatment-class 层面的总体证据。没有这一步，Step6 无法判断 shared core 与 dehydration-biased。它也不多余，因为它只生成类内 pooled evidence，不直接下最终标签。([Springer][10])

**Step6 不可缺少**
它是把中间统计对象真正转化为科学结论的步骤。没有这一步，你只有一堆 pooled effects，没有对主问题的回答。它也不多余，因为它只保留两个主终点，没有再开新主线。([ResearchGate][12])

**Step7 不可缺少**
你的主问题不只问“有没有 shared core / dehydration-biased”，还问“由谁调控、落在什么细胞语境”。没有 Step7，论文在机制解释上不完整。它也不多余，因为它不反向改写主标签，只解释 Step6 已固定结果。([Rutgers University][3])
